College Website

Status: Updating

Description:

I was given, for an individual project, a client brief to create a website for ‘Cantor College’. The project focuses on frontend development but does have a db connection to store the course information. Even though I only require the db connection for 2 of the pages all the pages are php files this is becuase of my use of include for my header and footer.

Toward the end of the project I used Lighthouse, a chrome tool, to audit each page I improved my website using the results of these audites. Currency Transfer Application

Status: Complete

Description:

This Currency Transfer Application (CTA) allows customers to store, exchange, and transfer money in different currencies. An API is connected to provide automatic exchange rate updates to the 161 major currencies used in 200 countries. There is also a db connection, hosted on a local server by xampp to store user information.

New customers create accounts at the register page they then can login to open up a ‘wallet’ page with an empty main (GBP) ‘wallet’. On this page the user creates and or removes ‘wallets’ of different currencies, once created the customer can start exchanging their money. The account page will allow for the update / change of personal details.

Customer’s can transfer money between their currency ‘wallets’ and can send funds to a friend's account if they enter the correct details into the relevant fields. Before transferring funds the customer will need to exchange the money, this is for the verification of the transaction; checks if there are sufficient funds, and if the user wants to increase or decrease the amount.


Languages: HTML, CSS, JS, PHP, SQL

Software Used: XAMPP, SQLite, PhpMyAdmin, GitHub, Boostrap, Font Awesome, an API

EER Calculator App

Status: Complete

Description:

A web-based applicaion of an Energy Effciency Rating (EER) calculator, myself and 4 others worked together with a client for this project. We ensured all interactions with the db was secure to protect both the user's personal details and the dependability of the application, the db stores user information, buildings, saved buildings, and residential buildings.

User accounts are of 2 types; tenants and landlords. The landlords can add buildings, enter data into the calculator, remove their buildings, update calculations, search and view other buildings. A tenant account can only view the buildings and their ratings as well as save them to the dashboard. On the account page a user's details can be updated.

The browse page has a search bar and filter fields, a tagged map, and a list of the buildings with their name, address, and rating. The user can search for a building name, an address, or building type. For the filter all buildings with the selected rating/s will be listed underneath the map. The results of any search or filter will be displayed on the map with a tag of the corresponding rating colour.

My team and I managed to meet the client's requirments within the set time frame, before we began action plans, roles, and responsibilities were written down to improve our collaboration. For maintenance the admin will review building information to ensure it is valid before making it visible to other users. In the future the client would like to add features if ideas for them come up like giving the user a way to compare building's. Our university lecturer acted as a client and we treated them as such.


Languages: HTML, CSS, JS, TS, PHP, SQL

Software Used: XAMPP, SQLite, PhpMyAdmin, GitHub, JSON
